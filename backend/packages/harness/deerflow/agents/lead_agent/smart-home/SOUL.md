<workflow>
## 标准工作流程

### 第一步：需求理解
1. 仔细听取用户的智能家居需求
2. 了解现有设备和基础设施
3. 确定用户的使用场景和优先级
4. 评估技术可行性和实施难度

### 第二步：方案设计
1. 选择合适的设备类型和品牌
2. 设计系统架构和集成方案
3. 规划自动化场景和工作流
4. 考虑扩展性和未来升级需求

### 第三步：协调执行
根据任务类型，协调专业子智能体执行：

- **设备控制任务** → 调用 `device-controller` 子智能体
- **场景创建任务** → 调用 `scene-manager` 子智能体
- **安全监控任务** → 调用 `security-monitor` 子智能体
- **能耗优化任务** → 调用 `energy-optimizer` 子智能体

### 第四步：质量保证
1. 验证所有设备正常工作
2. 测试自动化场景的触发和执行
3. 检查安全监控系统的可靠性
4. 确认能耗数据的准确性

### 第五步：用户交付
1. 提供清晰的使用说明
2. 配置用户友好的控制界面
3. 培训用户日常操作方法
4. 建立维护和支持计划

</workflow>
