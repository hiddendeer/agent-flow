# 宅力觉醒智能体 Backend 核心思想摘要

本文档总结了宅力觉醒智能体（Agent-Flow Backend）的系统架构及核心设计思想，帮助开发者快速把握整个后端的脉络体系。

## 1. 宏观架构与分层设计

整个系统由前端接入的 **Nginx（2026端口）** 充当统一反向代理，依据请求路径将流量分发到底层的两个主要服务进程：

- **/api/langgraph/*** 👉 **LangGraph Server（2024端口）**：专门管理 Agent 会话交互、状态机流转、子智能体编排和执行流式响应。
- **/api/* (其余接口)** 👉 **Gateway API（8001端口）**：基于 FastAPI，负责外延性的业务管理功能（例如模型配置、MCP 服务设置、技能管理、记忆查询以及工件/文件上传服务）。

在项目代码层面也做到了严格的**单向模块解耦**：
- **App 层 (`app/`)**：纯对外的业务接口层（如 HTTP API 和 IM 机器人交互），允许调用底层模块。
- **Harness 层 (`packages/harness/deerflow/`)**：封装着大模型和 Agent 流程运转的核心驱动框架，此层高度抽象，绝不能反向依赖 App 层，从而保证其极高复用性与作为单独 package 被发布的可能。

## 2. Lead Agent 中心大脑与全链条中间件 (Middleware Chain)

作为处理会话的心脏，**Lead Agent** 利用 LangGraph 进行节点间的交互，它支持灵活切换语言大模型（如支持视觉以及思考能力分配）。
它最具特点的设计是采用了**“中间件链模式”**。所有对话或任务生成执行前，均顺次执行包含下面典型模块在内的多达 12 层的中间件链条：
- `ThreadDataMiddleware`：创建线程级的上下文和本地化状态空间。
- `UploadsMiddleware`：将用户发送的新文件智能注入对话上下文。
- `SandboxMiddleware`：装载安全的沙盒执行环境。
- `MemoryMiddleware`：会话结束时的记忆数据排队过滤。
- `ClarificationMiddleware`：拦截大模型产生的中断问题，反向请求澄清并暂停任务流直至接手输入。

设计极为松耦合且职责单一，使得后续功能的拓展通过挂载新的中间件即可实现。

## 3. 沙盒与环境隔离机制 (Sandbox System)

为了让大模型放心、安全地操作本地或容器的文件与命令，系统引入每线程相互独立的设计：
- **资源完全隔离：** Mỗi一个线程对应分配独立的底层真实文件夹(`.deer-flow/threads/{id}`)，内含 Workspace、Uploads、Outputs 操作区。无论大模型如何覆写资源，都不会跨会话污染。
- **虚拟路径体系：** 大模型视角的绝对路径往往被挂载成特定的规范目录（例如 `/mnt/user-data/workspace`），该路径下发到底层时会被自动转译处理。
- **读写一致化工具：** 为了避免覆盖冲突，Agent 使用定制化的 `str_replace` 与文件操作相关工具，保障原子化更新。

## 4. “越用越懂你”的记忆系统 (Memory System)

架构内实现了自动学习与用户画像提取机制（Memory Update Pipeline）：
- **异步防抖、平滑更新：** 每次正常对话交流并不会直接触发缓慢的认知总结请求。而是流过提取队列并挂起，经过防抖期（比如最近30秒无新消息）后，再在后台统一合并送入 LLM 计算新认知。
- **逻辑分层的认知体系：** 模型提取的记忆包含：用户背景(工作与生活)、往期关注重点(History)和具有权重打分的知识库事实库(Facts，如用户常用语言、发现过哪些修正建议)。
- **精准的提示词融合：** 系统通过 tiktoken 精确评估 Token 开销，并在长久限制阈值内选取高优的关联记忆信息，顺滑注入下次发送请求的 System Prompt 中。

## 5. 任务分发网络与扩展系统

整个框架不局限于单一任务，设计了极强的对外连通与派生能力：
- **Subagents 子智能体池：** 当 Lead Agent 面临大规模或者并发操作（比如同时查询三个网站总结资料）时，它可以下发布置任务（Task）。后端的双线程池（调度池+执行池）最多支持3个异步任务同时开辟执行并保持状态长连接实时轮询反馈事件。
- **强大的 MCP (Model Context Protocol) 标准兼容：** 内层天生自带对最新 MCP 通信特性的整合处理，可以轻易挂载类似于标准化代码仓管理(Github MCP)或是云上企业系统的操作能力。
- **丰富完备的 Tool/Skills 集：** 基于 `config.yaml` 或直接利用外部文件夹(skills)扫描出的结构化系统文档实现配置性热插拔。

## 总结

宅力觉醒智能体的后台结合了 **LangGraph的稳定图流流转** 和 **FastAPI在对外业务能力提供上的便捷**。利用分层架构、强大的流式响应体系、独立严格的沙盒线程管理以及无痛防抖式的记忆提取管线，使得其本身具备极高的开发扩展上限，并保证智能体环境的绝对安全与个性化体验。
